# Порівняння продуктивності: Pandas vs NumPy для обчислення ковзного середнього

## Опис проєкту
Цей проєкт демонструє підходи до оптимізації продуктивності при роботі з великими наборами даних. Скрипт генерує часовий ряд із 100 мільйонів записів та порівнює швидкість і точність обчислення ковзного середнього (rolling mean) з вікном у 7 днів за допомогою двох підходів: вбудованого методу `Pandas` та векторизованої операції `NumPy`.

У сучасному аналізі даних продуктивність має критичне значення, особливо коли потрібно працювати з великими наборами даних. Великі дані вимагають швидкої обробки (Velocity) та оперують величезними масивами інформації (Volume). Однією з головних порад щодо оптимізації коду є використання векторизованих операцій (vectorized operations) замість стандартних циклів.

## Вимоги
Для запуску скрипта необхідні наступні бібліотеки:
* `pandas` — для створення DataFrame та базового обчислення.
* `numpy` — для швидких математичних операцій над масивами.

## Як працює скрипт

### 1. Генерація даних
Скрипт створює масив зі 100,000,000 рядків, що містить дві колонки:
* `date`: щоденні дати, починаючи з 2000-01-01.
* `revenue`: випадково згенеровані значення доходу від 100 до 5000.

### 2. Метод Pandas
Використовується стандартний ланцюжок методів `.rolling(window=7).mean()`. 
* **Переваги:** Висока читабельність коду, автоматична обробка крайових значень (NaN) та стабільність обчислень.
* **Недоліки:** Може бути повільнішим на дуже великих масивах через внутрішні перевірки (overhead) бібліотеки.

### 3. Метод NumPy
Використовується кумулятивна сума `np.cumsum()`. 
* **Алгоритм:** Знаходиться загальна сума елементів, а потім з неї віднімається та сама сума, але зміщена на розмір вікна. Результат ділиться на розмір вікна (7).
* **Переваги:** Максимальна швидкість завдяки векторизації операцій.
* **Недоліки:** Потребує ручного вирівнювання розмірів масивів (додавання NaN на початку) та може накопичувати мікроскопічні похибки через особливості арифметики з плаваючою комою (floating-point arithmetic).

### 4. Порівняння результатів
Наприкінці скрипт обчислює `max_difference` — максимальну абсолютну різницю між результатами обох методів. Це дозволяє переконатися, що оптимізований алгоритм NumPy працює коректно, а незначні розбіжності лежать у межах допустимої машинної похибки.
